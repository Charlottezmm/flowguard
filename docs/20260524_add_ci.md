# Add GitHub Actions CI

## 问题描述
- v0.1 已经完成并关闭 issues，但质量验证仍依赖本地手动命令。
- `main` 已启用基础保护规则，但还没有 required status checks。

## 解决方案
- 新增 GitHub Actions CI，在 pull request、`main` push 和手动触发时运行。
- 使用 Python 3.10 和 3.12 验证最低支持版本和当前本地主要版本。
- CI 显式创建 `.venv`，安装 `.[dev]`，运行 demo、验证四个 FlowGuard artifacts、运行测试和 compileall。

## 实施步骤
1. 新增 `.github/workflows/ci.yml`。
2. 在 CI 中执行 demo：
   ```bash
   PYTHONPATH=src .venv/bin/python examples/github_issue_triage/pipeline.py
   ```
3. 验证生成：
   ```text
   .flowguard/runs/latest/trace.json
   .flowguard/runs/latest/workflow_map.json
   .flowguard/runs/latest/agent_context.md
   .flowguard/runs/latest/outcome_report.html
   ```
4. 运行：
   ```bash
   PYTHONPATH=src .venv/bin/python -m pytest
   .venv/bin/python -m compileall src tests examples
   ```

## 测试验证
- 本地执行同等验证命令，确认 demo artifact、CLI、pytest 和 compileall 都通过。
- PR 合并后，在 GitHub 上确认 `CI / Python 3.10` 和 `CI / Python 3.12` checks 通过。

## 注意事项
- CI 不引入 lint/type-check 新工具，避免超出 v0.1 收尾范围。
- required status checks 应在 CI PR 合并且 checks 名称稳定后再启用。
- demo 中的 `issue.triage` 失败是预期的 silent failure 检测结果，不代表 CI 失败。
