# FlowGuard v0.2 Core

## 问题描述
- v0.1 能生成一次运行的修复上下文，但失败检查只有字符串，后续 golden compare、query API 和 MCP 都会被文本绑定。
- v0.1 只能检查 Python 对象的基础字段，不支持 workflow 常见的文件 artifact 和 response-like output。
- 没有本地 golden baseline，无法判断一次改动是否破坏了关键 run behavior。

## 解决方案
- 新增结构化 `checks` 字段，同时保留 v0.1 `failures` 字符串投影。
- 新增 file artifact checks：文件存在、非空、扩展名、合法 JSON。
- 新增 response-like checks：状态码范围、必填字段、非空列表、error envelope。
- 新增 `flowguard.query` 只读 API，集中读取 latest run、workflow map、agent context 和 run summary。
- 新增 golden baseline create/compare，忽略 timestamps、duration 和 input/output summaries 等不稳定字段。
- 新增实验性 read-only MCP stdio adapter，只暴露 latest status、failed step、workflow map、agent context。

## 实施步骤
1. 先用 TDD 增加 structured checks 和 runtime trace 写入测试。
2. 增加 file/API checks 的单元测试。
3. 增加 query API 和 golden baseline 测试。
4. 增加 CLI golden create/compare 测试。
5. 增加 MCP JSON-RPC smoke tests。
6. 更新 README、FlowGuard skill、版本号和 v0.2 planning 文档状态。

- `PYTHONPATH=src .venv/bin/python -m flowguard.cli golden create --workflow github_issue_triage --name default`
- `PYTHONPATH=src .venv/bin/python -m flowguard.cli golden compare --workflow github_issue_triage --name default`
- `.venv/bin/python -m compileall src tests examples`

## 注意事项
- `agent_context.md` 仍按 v0.1 schema 输出；结构化 checks 是 trace/query/golden 的数据层增强。
- MCP adapter 是 experimental 且只读，不执行 workflow、不写 artifact。
- Golden compare 默认不比较 input/output summary，避免真实 workflow 中的时间、随机 ID、API 顺序等噪声。
- File checks 从 step output 字段读取路径；相对路径按 workflow run root 解析，golden baseline 会归一化绝对路径前缀。
